# Простое веб-приложение с формой

Одностраничное веб-приложение с формой обратной связи. Данные сохраняются в файл на сервере.

## Установка и запуск локально

### 1. Установите зависимости

```bash
npm install
```

### 2. Запустите сервер

```bash
npm start
```

Сервер запустится на `http://localhost:3000`

### 3. Откройте приложение в браузере

Перейдите по адресу: http://localhost:3000

## Деплой на Vercel

Проект настроен для работы с Vercel serverless функциями.

### Важно о хранении данных на Vercel

⚠️ **В текущей версии данные хранятся в `/tmp`, который очищается между вызовами функций.**

Для production рекомендуется использовать:
- **Vercel KV** (key-value хранилище)
- **MongoDB Atlas** (бесплатный тариф)
- **Supabase** (бесплатный тариф)
- Другие облачные базы данных

### Как задеплоить:

1. Установите Vercel CLI (если еще не установлен):
```bash
npm i -g vercel
```

2. Задеплойте проект:
```bash
vercel
```

Или подключите репозиторий GitHub к Vercel через веб-интерфейс.

## Структура проекта

- `index.html` - главная страница с формой
- `style.css` - стили приложения
- `script.js` - клиентская логика
- `server.js` - сервер для локальной разработки
- `api/submit.js` - serverless функция для сохранения данных (Vercel)
- `api/submissions.js` - serverless функция для получения данных (Vercel)
- `package.json` - зависимости проекта
- `vercel.json` - конфигурация Vercel
- `data/submissions.json` - файл для хранения данных (локально)

## Функциональность

- Форма с тремя полями: имя, почта, комментарий
- Валидация полей
- Сохранение данных (локально в файл, на Vercel в /tmp)
- Состояние успеха после отправки
- Красивый современный интерфейс

## Просмотр сохраненных данных

### Локально:
Все данные сохраняются в файл `data/submissions.json` в формате JSON.

### На Vercel:
Данные хранятся в `/tmp/submissions.json` (временное хранилище).

Также можно получить данные через API:
- GET запрос на `https://ваш-домен.vercel.app/api/submissions`

Или в консоли браузера:
```javascript
getAllSubmissions()
```
